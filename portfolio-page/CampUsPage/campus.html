6인 팀 프로젝트 / 캠핑 통합 예약 웹어플리케이션

6명의 백엔드 개발자로 이뤄진 프로젝트에서, 전체 시스템을 담당하기 위해 풀스택 개발을 맡았습니다.

다른 팀원들은 HTML, CSS, JavaScript에 익숙하지 않았기 때문에, 프로젝트 시작 전에 기초적인 교육를 진행하고 나서 프로젝트를 진행했습니다.

이 프로젝트는 Spring Boot 연습용 프로젝트로, Spring의 구조와 동작 방식을 보다 쉽게 이해할 수 있는 SSR(서버 사이드 렌더링) 방식을 선택했습니다.

주로 기본적인 CRUD, 공공 API, Docker에 초점을 맞추고 개발을 진행했습니다.

Architecture

Experience

이 프로젝트에서 저는 풀스택 개발자로서 전체 프론트엔드 개발을 주도했으며, 로그인 및 상세 페이지의 백엔드 개발을 담당했습니다.

프론트엔드 작업 중에는 동료 개발자들이 HTML, CSS, JS 및 jQuery에 익숙하지 않아 이 부분에 대한 강의도 진행했습니다.

이를 통해 복습 기회가 생기면서 동시에 다른 팀원들에게 지식 전달이라는 측면에서 큰 즐거움을 느꼈습니다.

Docker를 처음 사용한 프로젝트여서 세팅하는 데 많은 시간이 소요되었고, Docker를 다루는 것이 예상보다 어려웠습니다.

Docker를 적용하는 과정에서 다른 부분들과의 충돌로 인해 많은 시간을 투자해야 했습니다.

docker-compose up 명령어를 실행했을때 port 오류가 발생했는데,

기존에 Mysql이 사용하던 포트와 docker가 사용할 포트 3306이 겹처서 발생하는 문제였습니다.

cmd에서 pid를 확인 후 taskkill /f /pid number로 프로세스를 종료하여 해결했습니다.

또한 Docker Desktop을 업데이트 하면 생기는 오류 역시 settings.json 파일을 삭제 후 재실행을 하여 해결했습니다.

또한, 캠핑 목록과 캠핑 상세 페이지에서는 공공 API를 처음 사용해보았는데,

이를 통해 API를 받아오고 어떻게 적용하는지에 대한 새로운 경험이 되었습니다.
